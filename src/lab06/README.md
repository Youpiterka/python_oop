# ЛР-6 — Generics и typing

## 1. Цель работы

Научиться:
- ставить аннотации типов в Python (модуль `typing`);
- делать обобщённые (generic) классы через `TypeVar` и `Generic`;
- использовать `Protocol` для структурной типизации.

## 2. Описание реализованных типов и контейнеров

Аннотации типов добавлены в существующие файлы прошлых лабораторных:
- `lab01` — класс `Character` с аннотациями (параметры, атрибуты, возвраты);
- `lab03` — `Warrior` и `Mage` с аннотациями. В базовый `Character`
  добавлены методы `display()` и `score()` для протоколов;
- `lab02` — не изменялся.

В `lab06/container.py`:
- `TypedCollection(Generic[T])` — generic-версия коллекции из ЛР-2.
  Методы `add`, `remove`, `get_all`, `__len__`, `__getitem__`,
  `remove_at`, `__iter__` с аннотациями. Дубликат не добавляется
  (проверка через `__eq__`).
- Задание на 4: методы `find`, `filter`, `map`. У `map` второй
  TypeVar `R`, поэтому тип результата может отличаться от типа элементов.
- Задание на 5: протоколы `Displayable` и `Scorable`. Классы из ЛР-3
  не наследуются от них, но имеют методы `display()` и `score()`,
  значит подходят под протоколы.

Использованные TypeVar: `T` (элементы коллекции), `R` (результат `map`),
`D` с `bound=Displayable`, `S` с `bound=Scorable`.

## 4. Вывод

- Аннотации типов делают код понятнее и помогают находить ошибки заранее.
- `Generic` и `TypeVar` позволяют написать одну коллекцию для любого типа.
- Второй TypeVar `R` нужен, чтобы `map` мог менять тип результата.
- `Protocol` — это интерфейс без наследования: важно, что у класса есть
  нужный метод, а не от кого он наследуется.
