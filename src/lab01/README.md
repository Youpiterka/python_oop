# Лабораторная работа №1 — Класс и инкапсуляция

Предметная область: вариант номер 6 Игровая логика


Реализован класс **Character**, описывающий персонажа игры с характеристиками здоровья, уровня, урона и опыта.

Класс поддерживает:
- создание объектов с проверкой данных;
- изменение характеристик через свойства;
- сравнение персонажей;
- атаку и получение урона;
- накопление опыта;
- контроль состояния персонажа (*жив / мёртв*).

# Класс `Character`
## Атрибут класса
- **max_level = 100** — максимальный уровень персонажа.

## Закрытые атрибуты экземпляра
- `_name` — имя персонажа.
- `_health` — количество здоровья.
- `_level` — уровень.
- `_damage` — наносимый урон.
- `_experience` — опыт.
- `_alive` — состояние персонажа.

---

# Конструктор
## `__init__(name, health, level, damage)`
Создаёт нового персонажа и выполняет валидацию входных данных.

### Проверяется:
- имя должно быть непустой строкой;
- здоровье > 0;
- уровень от 1 до `max_level`;
- урон ≥ 0.

---

# Методы валидации
## `_validate_name(value)`
Проверяет корректность имени.

## `_validate_health(value)`
Проверяет здоровье.

## `_validate_level(value)`
Проверяет диапазон уровня.

## `_validate_damage(value)`
Проверяет урон.

---

# Свойства (`@property`)
## Доступны для чтения:
- `name`
- `health`
- `level`
- `damage`
- `experience`

## Setter
### `damage.setter`
Позволяет изменять урон с повторной проверкой значения.


---

# Магические методы
## `__str__()`
Возвращает красивую строку для `print()`.

Пример вывода:
```text
V | lvl 1 | HP 100 | dmg 15 | alive
```

## `__repr__()`
Возвращает техническое представление объекта.

## `__eq__(other)`
Сравнивает двух персонажей по имени и уровню.

---

# Бизнес-методы
## `attack(other)`
Персонаж атакует другого персонажа и наносит урон.


## `take_damage(amount)`
Уменьшает здоровье.
Если здоровье стало `<= 0`, персонаж погибает.

## `gain_experience(exp)`
Добавляет опыт. При достижении 100 очков персонаж должен повысить уровень.
