Предметная область: вариант номер 6 Игровая логика

Что реализовано

Атрибуты
max_level = 100 — общий для всех персонажей потолок уровня. Остальные поля (_name, _health, _level, _damage) закрыты — доступ к ним только через свойства.

Валидация
Для каждого поля есть отдельный метод проверки. Если передать пустое имя, отрицательное здоровье или уровень вне диапазона — сразу вылетит ошибка с понятным текстом.
_validate_name, _validate_health, _validate_level, _validate_damage. 

Свойства @property
Геттеры есть у всех атрибутов. У damage дополнительно есть сеттер — можно менять урон после создания объекта.

Магические методы
__str__ выводит персонажа в читаемом виде, __repr__ — в техническом, __eq__ сравнивает двух персонажей по имени и уровню.

Бизнес-методы
attack — атакует другого персонажа; мёртвый атаковать не может
take_damage — получает урон, при HP ≤ 0 персонаж умирает
gain_experience — копит опыт, при 100 единицах автоматически повышает уровень
Логическое состояние
Флаг _alive меняется при получении фатального урона и блокирует возможность атаковать — попытка вызовет RuntimeError.
